# Вопросы для самопроверки

1. В чем главное концептуальное отличие GraphQL от REST?
REST - каждый эндпоинт отдаёт строго определённый набор данных. Клиент получает всё, даже если нужно только одно поле.
GraphQL - один эндпоинт (/graphql), клиент пишет запрос и получает ровно запрошенные поля.
GraphQL передаёт контроль над данными клиенту, REST жёстко определяет структуру ответа на сервере.
2. Что такое Schema Definition Language (SDL)?
SDL - это язык описания схемы GraphQL. Чертёж, по которому строится API.
3. Чем Query отличается от Mutation? 
Query - для чтения данных (как GET)
Mutation - для изменения данных (как POST/PUT/DELETE)
Могут ли они выполняться параллельно?
Query - да, mutation - нет, они выполняются строго по очереди (чтобы избежать конфликтов при изменении данных)
4. Что такое "резолвер" (resolver) и кто его вызывает?
Резолвер — это функция, которая знает, где взять данные для конкретного поля. Вызывается GraphQL-движком (например, Strawberry)
5. Как решается проблема N+1 запросов в GraphQL (dataloader)?
Проблема N+1 возникает, когда GraphQL делает один запрос для получения списка объектов и затем для каждого объекта отдельный запрос для связанных данных. DataLoader работает как умный сборщик:
Собирает все запросы на получение данных за один цикл событий
Группирует их 
Выполняет один общий запрос к базе
Возвращает результаты каждому резолверу в правильном порядке
6. Почему в GraphQL сложно реализовать кэширование на уровне HTTP (как в REST)?
В отличии от REST, GraphQL сложно кэшировать на уровне HTTP, так как все запросы идут на один URL (/graphql) и тело запроса - это не URL, а сложный JSON
