# Project Code
events-s16

## бизнес-процесс
Регистрация участника на платное мероприятие:

# состояния
NEW - заявка создана, ожидает оплаты
PAID - взнос оплачен, ожидает подтверждения
DONE - участие подтверждено, билет выдан
CANCELLED - регистрация отменена

## описание шагов
Текущее состояние + Событие -> Следующее состояние 

NEW + PAY_OK -> PAID                
NEW + PAY_FAIL -> CANCELLED         
PAID + complete -> DONE              
PAID + cancel -> CANCELLED           
PAID + PAY_FAIL -> CANCELLED           

## компенсации
- При ошибке оплаты (PAY_FAIL) → CANCELLED удалить заявку на регистрацию
- При отмене после оплаты (cancel) → CANCELLED вернуть деньги и удалить заявку
