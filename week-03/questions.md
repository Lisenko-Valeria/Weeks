# Вопросы для самопроверки

1. В чем разница между Forward Proxy (обычный прокси) и Reverse Proxy (обратный прокси)?
Forward Proxy (Обычный прокси):
-Прокси находится перед клиентом
-Клиент знает о прокси и настраивает его в своем приложении
-Скрывает клиента от сервера
Reverse Proxy (Обратный прокси):
-Прокси находится перед серверами
-Клиент не знает о прокси (думает, что общается напрямую с сервером)
-Скрывает серверы от клиента
2. Что такое Upstream в терминологии Nginx?
Upstream — это "настоящий" сервис, куда прокси пересылает запрос.
3. Зачем скрывать реальные IP-адреса сервисов от клиента?
Безопасность. Можно закрыть прямые порты микросервисов фаерволом, оставив только Gateway.
Единый домен.
SSL/TLS. Сертификаты можно настроить только на Gateway, а внутри защищенной сети ходить по простому HTTP.
Балансировка. Gateway может размазывать нагрузку между 10 копиями одного сервиса.
4. Какие еще задачи, кроме маршрутизации, может выполнять API Gateway (назовите хотя бы 2)?
Трансформация протоколов( например REST → gRPC)
Rate Limiting (Ограничение запросов)
5. Что произойдет, если Gateway упадет?
Все сервисы становятся недоступными для внешних клиентов
Как можно от этого защититься?
Настроить 2 экземпляра Gateway
